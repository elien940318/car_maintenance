# 시각화 · Acceptance Criteria (EARS)

> 01_planning: 요구사항을 검증 가능한 EARS로 정제. 이 명세가 가드레일.
> 반응형 분기: 모바일 (< 640px) / 태블릿+ (≥ 640px)

---

## 배경

- **모바일**: 티켓 카드 목록 단일 뷰. 색상으로 긴급도 표현. 알림 섹션 없음.
- **태블릿+**: 간트 차트 / 목록 테이블 탭 전환. 알림 카드 섹션 유지.
- 항목 탭·클릭 시 부품 상세 패널 표시 (교환완료 입력 포함).

---

## AC (Acceptance Criteria)

### 공통

**AC-VZ1** When 뷰포트 너비가 640px 미만일 때 (모바일),
the system shall 정비 항목을 **긴급도 순**으로 정렬하여 단일 플랫 목록으로 표시한다.
정렬 순서: urgent → soon → ok → unknown → chain.
같은 상태 내에서는 daysRemaining 오름차순 (초과 일수가 많은 항목이 상단).
카테고리 섹션 레이블은 표시하지 않는다.

> 태블릿+ (≥ 640px) 간트 차트·목록 테이블은 카테고리 섹션으로 그룹화한다 (VZ11·VZ17 참조).

**AC-VZ2** The system shall 차량의 연료에 적용되지 않는 부품은 목록에서 제외한다.
(예: EV → 엔진오일 표시 안 함, 가솔린 → 글로우플러그 표시 안 함)
- 적용 여부 기준은 `MaintenancePartMaster.applicable_fuel_codes` (단일 진실원, #11)

---

### 모바일 — 티켓 카드 목록 (< 640px)

**AC-VZ3** When 뷰포트 너비가 640px 미만일 때,
the system shall 간트 차트와 탭 없이 **티켓 카드 목록**을 단일 뷰로 표시한다.

**AC-VZ4** The system shall 각 티켓 카드에 아래 정보를 표시한다:
- 1행 좌: 부품명 (상태 색상) + 차량 전용 태그 또는 `[CHAIN]` 태그
- 1행 우: D-day 숫자 + 상태 아이콘·텍스트 (상태 색상)
- 2행 좌: **다음 교환 예정일** (큰 텍스트, `var(--text)`)
- 2행 우하단: 교환 주기 (`Nkm마다` 또는 `N개월마다`, 작은 muted 텍스트)
- chain 항목: 날짜·주기 행 생략, 1행 우에 "교체 불필요" 표시
- unknown 항목: 2행에 "계산 불가" (muted)
- 날짜·주기 텍스트에 카테고리 색상 미사용 (UI 기본색 통일)

**AC-VZ5** The system shall 티켓 카드의 긴급도를 **카드 테두리(stroke)와 부품명 텍스트 색**으로 표현한다. 배경 tint는 사용하지 않으며, 카드 배경은 `--bg2` 중립색으로 통일한다.

| 상태 | 카드 테두리(stroke) | 부품명 텍스트 |
|------|-------------------|-------------|
| urgent (< 90일 또는 초과) | `#f87171` (rose) | rose |
| soon (90~179일) | `#fbbf24` (amber) | amber |
| ok (≥ 180일) | `#22c55e` (green) | text(기본) |
| chain (교환 불필요) | `#38bdf8` (cyan) | cyan |
| unknown (계산 불가) | `#6b7a99` (muted) | muted |

**AC-VZ6** When `isChain=true`인 항목일 때,
the system shall 카드에 "교체 불필요 · 모니터링만 필요" 텍스트를 표시하고
날짜 관련 행을 생략한다.

**AC-VZ7** The system shall 모바일 뷰에서 알림 카드 섹션을 표시하지 않는다.
(긴급도는 티켓 카드 색상으로 충분히 전달됨)

---

### 태블릿+ — 탭 뷰 (≥ 640px)

**AC-VZ8** When 뷰포트 너비가 640px 이상일 때,
the system shall 간트 차트 뷰와 목록 보기 뷰를 탭으로 전환할 수 있어야 한다.

**AC-VZ9** The system shall urgent 및 soon 상태 항목을 뷰 상단에 알림 카드로 집계 표시하고,
예정일 오름차순으로 정렬한다. (태블릿+ 전용)

**AC-VZ10** When 알림 카드가 없을 때, the system shall 알림 영역을 표시하지 않는다.

#### 간트 차트

**AC-VZ11** The system shall 3년 타임라인(월 단위)의 간트 차트를 표시한다.

**AC-VZ12** The system shall 각 정비 항목에 대해 아래 바(bar)를 표시한다:
- **완료 바**: 최근 교환일 ~ 오늘 (낮은 불투명도)
- **다음 교환 바**: 오늘 ~ 다음 교환 예정일
- **미래 바**: 다음 교환 예정일 ~ 그 다음 교환 예정일 (더 낮은 불투명도)

**AC-VZ13** The system shall 오늘 날짜를 수직선(TODAY line)으로 표시한다.

**AC-VZ14** The system shall 간트 차트 초기 렌더링 시 TODAY 라인이 가시 영역 좌측 1/4 지점에 위치하도록 자동 스크롤한다. 페이지 재방문·데이터 갱신 후에도 동일하게 유지한다.

**AC-VZ15** When `isChain=true`인 항목일 때,
the system shall 교환 바 대신 "교체 불필요" 패턴 배너를 표시한다.

**AC-VZ16** The system shall 다음 교환 예정월 핀(날짜 레이블)을 해당 바 위에 표시한다.

#### 목록 테이블

**AC-VZ17** The system shall 목록 뷰에서 각 항목에 대해 아래 컬럼을 표시한다:
항목명 / 주기 / 최근 교환 / 다음 교환 예정일 / 예상 km / 상태

**AC-VZ18** The system shall 상태(urgent/soon/ok/chain/unknown)를 색상 배지로 표시한다.

---

### 부품 상세 패널 (공통)

**AC-VZ19** When 사용자가 정비 항목(티켓 카드, 간트 바, 목록 행, 알림 카드)을 탭·클릭하면,
the system shall 아래 정보가 담긴 상세 패널을 표시한다:
- 부품명·서브태그·D-day (또는 초과 일수)
- 부품 구조 SVG 일러스트
- 부품 역할 설명
- 교환 통계 (주기 / 다음 교환일 / 예상 km / 남은 날짜)
- 정비 팁
- **교환완료 기록 입력 영역** (하단 고정, AC-M12 연동)

**AC-VZ20** When 뷰포트 너비가 640px 미만이면 상세 패널을 **바텀 시트**로 표시한다:
- 하단 슬라이드업, max-height 75vh, 내부 스크롤 가능
- 상단 드래그 핸들 표시
- 핸들 스와이프 다운 또는 오버레이 탭으로 닫기

**AC-VZ21** When 뷰포트 너비가 640px 이상이면 상세 패널을 **우측 사이드 패널**로 표시한다:
- 오른쪽 슬라이드-인 (0.28s cubic-bezier)
- 배경 오버레이 (blur 2px)
- 오버레이 클릭 또는 ✕ 버튼으로 닫기

**AC-VZ22** The system shall 교환완료 입력 영역을 패널·시트 하단에 항상 고정 노출한다:
- 교환 날짜 (기본값: 오늘)
- 교환 km (기본값: 차량 현재 km)
- 메모 (선택)
- 저장 버튼 → AC-M13 실행 (별도 모달 없이 패널 내 인라인 처리)

**AC-VZ23** When 패널이 열려 있는 동안 배경 오버레이를 탭하거나 ✕·핸들 스와이프 다운을 수행하면,
the system shall 패널·바텀 시트를 닫는다.
